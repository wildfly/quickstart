= helloworld-mdb: Helloworld Using an MDB (Message-Driven Bean)
:author: Serge Pagop, Andy Taylor, Jeff Mesnil
:level: Intermediate
:technologies: JMS, EJB, MDB
:openshift: true

[abstract]
The `helloworld-mdb` quickstart uses JMS and EJB Message-Driven Bean (MDB) to create and deploy JMS topic and queue resources in {productName}.

:standalone-server-type: full
:archiveType: war

== What is it?

The `helloworld-mdb` quickstart demonstrates the use of JMS and EJB Message-Driven Bean in {productNameFull} with a remote {BrokerProductName} broker.

This project creates two JMS resources:

* A queue named `HELLOWORLDMDBQueue` bound in JNDI as `java:/queue/HELLOWORLDMDBQueue`
* A topic named `HELLOWORLDMDBTopic` bound in JNDI as `java:/topic/HELLOWORLDMDBTopic`

== Running the {BrokerProductName} broker locally
You can run {BrokerProductName} broker locally using docker or podman:
[source,options="nowrap"]
----
podman run -e AMQ_USER=admin -e AMQ_PASSWORD=admin -p61616:61616 -p8161:8161 -e AMQ_EXTRA_ARGS="--http-host 0.0.0.0 --host 0.0.0.0 --relax-jolokia" --name artemis quay.io/artemiscloud/activemq-artemis-broker:latest
----

You can access the console of {BrokerProductName} at the URL: http://localhost:8161/console with admin/admin to authenticate.

== Access the Application

The application will be running at the following URL: http://localhost:8080/{artifactId}/ and will send some messages to the queue.

To send messages to the topic, use the following URL: http://localhost:8080/{artifactId}/HelloWorldMDBServletClient?topic
